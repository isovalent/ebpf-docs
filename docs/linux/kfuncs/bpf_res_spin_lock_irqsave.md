# KFunc `bpf_res_spin_lock_irqsave`

<!-- [FEATURE_TAG](bpf_res_spin_lock_irqsave) -->
[:octicons-tag-24: v6.15](https://github.com/torvalds/linux/commit/97eb35f3ad42de1c932ef1f7e2f0044d4fca35f4)
<!-- [/FEATURE_TAG] -->

Takes a Resilient Queued Spin Lock (`rqspinlock`). And disabled IRQs.

## Definition

A variant of [`bpf_res_spin_lock`](bpf_res_spin_lock.md) which disables IRQs. See [`bpf_local_irq_save`](bpf_local_irq_save.md) for IRQ save semantics. [`bpf_res_spin_unlock_irqrestore`](bpf_res_spin_unlock_irqrestore.md) is to be used to release the lock.

**Returns**

A conditional result, depending on whether the lock was acquired (`NULL` is returned when lock acquisition succeeds, non-NULL upon failure). The memory pointed to by the returned pointer upon failure can be dereferenced after the `NULL` check to obtain the error code.

Preemption and IRQs are disabled upon successful lock acquisition. 

**Signature**

<!-- [KFUNC_DEF] -->
`#!c int bpf_res_spin_lock_irqsave(struct bpf_res_spin_lock *lock, long unsigned int *flags__irq_flag)`

!!! note
	The pointer returned by the kfunc may be NULL. Hence, it forces the user to do a NULL check on the pointer returned 
	from the kfunc before making use of it (dereferencing or passing to another helper).
<!-- [/KFUNC_DEF] -->

## Usage

See [`bpf_res_spin_lock`](bpf_res_spin_lock.md) for details.
